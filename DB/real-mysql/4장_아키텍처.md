## MySQL 엔진 아키텍처
Mysql 엔진 -> 스토리지 엔진(innoDB, MyISAM) -> 디스크
Mysql 엔진: SQL 파서 -> SQL 옵티마이저 -> SQL 실행기


포그라운드 쓰레드(사용자 쓰레드): 사용자의 요청을 처리
백그라운드 쓰레드: 디스크 기록

## inno DB
mysql 의 스토리지 엔진 중 하나로 레코드 기반의 잠금을 제공하여, 높은 동시성 처리가 가능하다.

### 프라이머리 키에 의한 클러스터링
기본 키에 의한 클러스터링 인덱스란 데이터들이 디스크에서 기본 키에 의해 정렬되어 있다는 것이다.
기본 키 인덱스 트리에서 각 리프 노드는 레코드의 디스크 주소를 가리키는 것이 아니라 바로 데이터 그 자체를 가리킨다.
세컨더리 인덱스는 레코드 주소가 아닌 기본 키 값을 갖는다.
